# 프로젝트 격리 가이드

## 개요

이 프로젝트는 다른 Next.js 프로젝트와 동시에 실행될 수 있도록 격리 설정이 되어 있습니다.

## 포트 설정

- **개발 서버**: 포트 3001 사용
- **프로덕션 서버**: 포트 3001 사용

다른 프로젝트(예: habitree_v3.1)가 포트 3000을 사용 중이므로, 이 프로젝트는 3001을 사용합니다.

## 프로젝트 격리 방법

### 1. 포트 변경

`package.json`의 스크립트에서 포트를 명시적으로 지정:

```json
{
  "scripts": {
    "dev": "next dev -p 3001",
    "start": "next start -p 3001"
  }
}
```

### 2. 빌드 캐시 격리

각 프로젝트는 자체 `.next` 디렉토리를 가지므로 자동으로 격리됩니다.

### 3. 환경 변수 격리

프로젝트별 `.env.local` 파일을 사용하여 환경 변수를 격리합니다.

## 실행 방법

```bash
# 개발 서버 실행 (포트 3001)
npm run dev

# 브라우저에서 접속
# http://localhost:3001
```

## 주의사항

1. **포트 충돌 방지**: 다른 프로젝트와 포트가 겹치지 않도록 확인
2. **캐시 정리**: 문제 발생 시 `.next` 폴더 삭제 후 재빌드
3. **환경 변수**: 프로젝트별로 `.env.local` 파일 관리

## 문제 해결

### 포트가 이미 사용 중인 경우

```bash
# Windows에서 포트 사용 확인
netstat -ano | findstr :3001

# 프로세스 종료 (PID 확인 후)
taskkill /PID <PID> /F
```

### 빌드 캐시 문제

```bash
# .next 폴더 삭제
Remove-Item -Recurse -Force .next

# 재빌드
npm run dev
```
