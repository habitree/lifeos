# 브라우저 테스트 가이드

**버전**: v0.1  
**작성일**: 2025-01-27

---

## 테스트 준비

### 1. 개발 서버 실행

```bash
npm run dev
```

서버가 `http://localhost:3001`에서 실행됩니다.

### 2. 브라우저 열기

- Chrome, Firefox, Safari, Edge 등 최신 브라우저 사용
- 개발자 도구 열기 (F12 또는 Cmd+Option+I)

## 테스트 시나리오

### 시나리오 1: 첫 사용자 플로우

1. 앱 접속
2. Home 페이지에서 Baseline 확인
3. Baseline 토글 (수면 ON)
4. Daily Log 페이지로 이동
5. 오늘의 기록 작성 및 저장
6. Phase 페이지로 이동
7. Phase 변경 (Phase 2로 변경)
8. Home 페이지로 돌아가서 변경사항 확인

**예상 결과**: 모든 변경사항이 저장되고 표시됨

---

### 시나리오 2: Reset 플로우

1. Home 페이지에서 모든 Baseline을 ON으로 설정
2. Daily Log에 기록 작성
3. Reset 버튼 클릭
4. 확인 메시지 확인
5. Reset 실행

**예상 결과**: 
- Baseline 체크가 모두 OFF로 변경
- 오늘의 Daily Log가 제거됨
- Baseline 값은 유지됨

---

### 시나리오 3: 오프라인 → 온라인 플로우

1. 오프라인 모드 활성화
2. Baseline 토글
3. Daily Log 저장
4. 온라인 모드 복구
5. 자동 동기화 대기 (30초)

**예상 결과**: 
- 오프라인에서도 데이터 저장됨
- 온라인 복구 시 자동 동기화됨
- Supabase에 데이터 반영됨
