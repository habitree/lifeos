# LIFE OS APP v0.1  
## 기준으로 돌아오는 루틴 앱 – 개인용 MVP 설계 문서

---

## 0. 이 앱을 만드는 이유

이 앱은 목표를 달성하기 위한 앱이 아니다.  
이 앱은 **무너져도 다시 기준으로 돌아오기 위한 앱**이다.

- 연속성보다 복귀
- 성과보다 기준
- 관리보다 회복

이 앱은 “잘 사는 사람”을 만들지 않는다.  
**“다시 사는 사람”을 만든다.**

---

## 1. 앱의 핵심 철학 (Non-negotiable)

1. Baseline이 전부다
2. 확장은 옵션이고, 복귀는 필수다
3. 기록은 평가가 아니라 좌표다
4. 실패라는 개념은 존재하지 않는다
5. 오늘의 성공은 ‘기준으로 돌아왔는가’다

---

## 2. 핵심 개념 정의

### 2.1 Baseline (절대 기준)

사용자는 딱 **3개의 절대 기준**만 가진다.

예시:
- 수면: 22:00–05:00
- 이동: 걷기/러닝 1km 이상
- 기록: 3줄

특징:
- 점수 없음
- 연속일 없음
- 달성 여부는 ON / OFF 뿐

---

### 2.2 Phase System (상태 기반 구조)

Phase는 성장 단계가 아니라 **현재 상태**를 나타낸다.

- Phase 1: Baseline
- Phase 2: Stability
- Phase 3: Growth
- Phase 4: Identity

원칙:
- Phase는 자동으로 상승하지 않는다
- 사용자가 직접 선택한다
- 무너졌다고 Phase가 내려가지 않는다
- “나는 지금 Phase 1로 돌아간다”는 선택을 존중한다

---

### 2.3 Reset 개념 (이 앱의 차별점)

Reset은 실패가 아니다.  
Reset은 **복귀 선언**이다.

Reset 버튼을 누르면:
- 오늘의 목표는 Baseline 3개만 남는다
- 나머지 모든 목표/확장 기능은 숨겨진다
- 메시지:  
  > “오늘은 돌아오기만 하면 된다”

---

## 3. 핵심 기능 구조

### 3.1 Home 화면 (메인)

구성 요소:
- 오늘 날짜
- Baseline 3개 (ON / OFF)
- 🔄 Reset Today 버튼
- 현재 Phase 표시

금지 요소:
- 점수
- 연속일
- 랭킹
- 통계

---

### 3.2 Daily Log 화면

입력 항목:
- 오늘의 기준 한 줄 (필수)
- 몸 상태 (좋음 / 보통 / 무거움)
- 자유 메모 (선택)

특징:
- 빈 기록 저장 가능
- 체크 못 해도 저장 가능
- “안 썼다”는 실패 개념 없음

---

### 3.3 Phase 화면

표시 내용:
- 현재 Phase
- Phase 설명 문구

기능:
- Phase 수동 변경
- “지금 나는 이 상태다”를 선언하는 공간

---

## 4. MVP 범위 (반드시 지킬 것)

### MVP 화면은 단 3개

1. Home
2. Daily Log
3. Phase

MVP에서는 절대 추가하지 말 것:
- 통계 그래프
- 목표 관리
- 알림 폭탄
- SNS 기능

이 앱은 **조용해야 한다**.

---

## 5. 데이터 구조 개요 (개념 수준)

### User
- id
- created_at
- current_phase

### Baseline
- sleep (string)
- movement (number)
- record (string)

### DailyLog
- date
- baseline_check (boolean x3)
- one_line (string)
- body_state (enum)
- memo (text)

---

## 6. Local-first 원칙

- 모든 기록은 로컬 저장 우선
- 서버는 백업 역할
- 로그인 없이도 사용 가능
- 데이터 소유권은 사용자에게 있음

> 내 삶의 기준은 서버 장애와 함께 사라지지 않는다.

---

## 7. 고도화 로드맵 (내가 쓰다가 확장)

### 7.1 개인 고도화
- 주간 회고
- Baseline 변경 히스토리
- “이번 주 나는 몇 번 돌아왔는가”

### 7.2 서비스화 가능 지점
- Phase 상태 공유 (숫자 없음)
- “나는 지금 회복 중입니다” 같은 상태 메시지

### 7.3 철학 유지 조건
- 연속일 기능 추가 시 → 앱 정체성 붕괴
- 점수화 시 → 이 앱은 폐기 대상

---

## 8. 이 앱이 지키는 문장

> 이 앱은 나를 밀어붙이지 않는다.  
> 이 앱은 내가 돌아올 자리를 남겨둔다.

---

## 9. 개발 중 나에게 남기는 경고

- 기능을 늘리고 싶어질 때 → Baseline 문서를 다시 읽어라
- 남에게 보여주고 싶어질 때 → 이 앱은 먼저 나를 위한 것이다
- 성과가 보이고 싶어질 때 → 성과는 삶이 증명한다
